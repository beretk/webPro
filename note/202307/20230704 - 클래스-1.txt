<접근제한과 static, 패키지>

1. 멤버변수 선언
 1)접근제한자(클래스의 데이타나 메스드에 대하여 드른 클래스로부터 접근을 제한 하는것.)
   -public : 다른 모든 클래스에서 사용 가능(접근을 제한하지 않음)
   -protected : 해당 클래스와 동일한 패키지에 있거나 상속받은 클래스일 경우에만 사용 가능
   -default : 접근제어자를 명시하지 않은 경우의 디폴드 접근제어자. 
            (같은 패키지내의 클래스들은 public 권한을 갖고 접근가능)
   -private : 해당 클래스만이 이 멤버를 사용할 수 있다. 외부객체에서는 접근 불가

 2)static(클래스 변수=static변수) : 정적변수,공적변수
  :객체들끼리 공유하고 싶은 정보가 있을때 사용(깍두기)
  -클래스 변수는 클래스로 부터 생성된 객체들의 수와 관계없이 하나만 생성
  -한 클래스로 부터 생성된 모든 객체들은 클래스 변수 공유
  -클래스 변수는 클래스 이름을 통하여 접근
   *단점:가비지콜렉터의 관리 밖에 있으므로 메모리 항상 상주(메모리 부하)
 3)final(종단변수)
  :static변수로 사용되는 대표예로 상수(변하지 않는수/ final변수)로 정의하고 사용

2. 패키지의 이해
 -접근 범위 안에서효과적으로 호출하기 위해 사용
 -관련있는 클래스들을 패키지로 그룹지어 관리 하는것(클래스를 모아 놓은 폴더)

*패키지\클래스를 만든후 서로 다른 클래스를 사용하고자 할때는 반드시 import 사용
 (이클립스는 자동으로 import)
*스태틱 메소드는 필요할때만
*스태틱 변수는 공유할때
