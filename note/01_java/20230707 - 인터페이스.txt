<인터페이스>

1.인터페이스 란?
 -작업명세서(작업지시서) - "앞으로 이렇게 만들어 " 라고 표현해 놓은것
 :실제 구현된 것이 전혀 없는 기본 설계도
 :객체를 생성할수 없고, 클래스 작성에 도움을 줄 목적으로 사용
 :미리 정해진 규칙에 맞게 구현하도록 표준을 제시
 :추상메소드와 상수만을 멤버로 가질수 있음
 
 -다형성 가능
  :하나의 객체를 다양하게 많은 type로 만들수 있음
  :하나의 interface를 사용하여 다양한 구현방법 제공
  :하나의 클래스나 함수가 다양하게 동작

 -다중상속
 :클래스에서 클래스  상속 : 단일상속(O),다중상속(X) 
 :인터페이스 에서 클래스(implement) : 단일구현, 다중구현
 :인터페이스 에서 인터페이스 상속 : 단일상속, 다중상속

2.인터페이스의 문법
  -Class 대신 interface 예약어 사용(클래스와 비슷함)
  -추상메소드와 상수만 존재(실제 구현된 기능 없음)
  -private불가
  :상수나 메소드 만들때 private 접근 제한자 불가
 -변수 타입으로만 사용됨(객체를 생성할수 없음)
 -구현은 Implement 되는 클래스에서 함

3.장점
 -개발시간 단축
 -표준화
 -관계 없는 클래스들에게 관계 맺어줌
 -독립적인 프로그래밍 가능
