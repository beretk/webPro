<추상클래스>

1.추상클래스란?
-추상적으로 선언된 객체를 사용할 사용자가  꼭 overriding 강요

2.추상클래스 문법
-abstract(추상클래스 및 추상 메소드 선언 예약어)이용
-추상클래스에는 하나 이상의 추상메소드 포함(정의만 하고 구현 하지않음)
-메소드의 선언부만 존재. 실행부(구현부)는 없음
-실제 구현은 상속받은 클래스에서 함
:기능은 자식 클래스에게 위임(추상클래스에서 정의된 추상적인 기능은 하위 클래스 에서 상세 구현)
-클래스의 프레임만 구성.직접 객체 생성 불가능(abstract는 인스턴스화를 금지하는 키워드)

3.추상클래스의 필요성(강제성이 필요할때 사용)
-추상적인 정의만 해놓고 자식클래스에서 강제로 무엇인가 꼭 재정의 해서 사용해야 할 때

4.final(제한자)
-클래스 앞에 붙일때 : 상속금지
-멤버 메소드 앞에 붙일때:오버라이딩 금지
-멤버 변수 앞에 붙일때:변경금지(상수화 됨)
